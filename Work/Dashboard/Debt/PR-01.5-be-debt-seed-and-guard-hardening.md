## DB / Migration Expectations

No production migration.

This project is still in active development and has no production debt data.
Update the init schema and seed scripts directly. The accepted shape is:

- `Debt.Status` values: `active`, `paidOff`, `archived`, `deleted`
- `BudgetMonthDebt.ParticipationStatus` values: `included`, `notIncluded`,
  `removed`
- no fixture should insert `Debt.Status = 'closed'`
- no new logic should treat legacy `BudgetMonthDebt.Status = 'closed'` as the
  source of truth

If legacy `BudgetMonthDebt.Status` remains for compatibility, seed it only as a
compatibility mirror, not as the modeled participation state.

## DTO / API Contract

No public API shape change is expected.

Existing planned-payment patch endpoints keep their current routes and request
DTOs. Behavior changes only by rejecting invalid lifecycle/participation states
consistently.

## Validation Rules

For both single and bulk planned-payment patch:

- open budget month required
- row must exist
- row must not be legacy deleted
- row `ParticipationStatus` must be `included`
- `ParticipationStatus = 'removed'` should return the precise removed-row error
- `ParticipationStatus = 'notIncluded'` should return the precise not-included
  error
- source lifecycle must be `active` when a source exists
- source lifecycle `paidOff`, `archived`, or `deleted` must reject mutation
- plan scopes still require `SourceDebtId`

Default editor row reads:

- include `included`
- include `notIncluded` if that is part of the current PR 1 read contract
- exclude `removed` unless explicitly requesting deleted/removed rows

Wizard/materialization:

- wizard-created debts are source debts with lifecycle `active`
- materialized month debts start as `included`
- no wizard happy-path fixture should create paid-off, archived, deleted, or
  removed rows by accident

## Audit / History Behavior

No new audit table or event type is introduced.

Existing planned-payment audit behavior must remain unchanged for valid
active/included mutations. Rejected bulk mutations must not write partial audit
events.
